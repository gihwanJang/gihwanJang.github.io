4. 배열기반의 정렬 리스트(SortedArrayList)

배열기반의 리스트는 자바에서 java.util.ArrayList 통해 사용할 수 있지만 따로 정렬리스트를 제공하지는 않습니다.
정렬리스트가 필요한 경우 ArrayList.sort를 사용하거나 직접 정렬리스트를 구현하여야 합니다.
전 post에서는 비정렬을 알아보았으니 이번 post에서는 정렬 리스트를 알아보고 정렬과 비정렬에서 어떤 차이점이 있는지 살펴보겠습니다.
코드는 변경되거나 추가 한 것만을 추가 설명하겠습니다.

(코드) 

import java.util.Iterator;

class ArrayList<T extends Comparable<T>> implements Iterable<T>{
	private Object[] arr = null;
	private int capacity;
	private int size;
	
	private class ListIterator implements Iterator<T>{
		private  int cursor =0;
		@Override
		public boolean hasNext() {
			return cursor<size;
		}
		@Override
		public T next() {
			return (T)arr[cursor++];
		}
	}
	
	public ArrayList(){
		this.capacity = 10;
		this.arr = new Object[this.capacity];
		this.size = 0;
	}
	public ArrayList(Object... arr){
		this.capacity = arr.length*2;
		this.arr = new Object[this.capacity];
		for(int i=0; i<arr.length; ++i) {
			this.add(arr[i]);
		}
		this.size = arr.length;
	}
	
	public boolean isEmpty() {
		if(this.size==0)return true;
		return false;
	}
	
	public boolean isFull() {
		if(this.size==this.capacity)return true;
		return false;
	}
	
	public int size() {
		return this.size;
	}
	
	public void clear() {
		this.size=0;
	}
	
	public boolean contains(Object o) {
		int index = search(o);
		if(this.arr[index]==o)return true;
		return false;
	}
	
	public int indexOf(Object o) {
		int index = search(o);
		if(this.arr[index]==o)return index;
		return -1;
	}
	
	public void add(Object o) {
		if(this.isFull())this.increaseCapacity();
		if(isEmpty() || ((T)this.arr[this.size-1]).compareTo((T)o)<=0)
			this.arr[size]=o;
		else{
			int insertLoc=search(o);
			moveRight(insertLoc);
			arr[insertLoc]=o;
		}
		++size;
	}

	public void popFront() {
		if(this.isEmpty()) return;
		this.moveLeft(0);
		--this.size;
	}
	
	public Object peekFront() {
		return this.arr[0];
	}
	
	public void popBack() {
		if(this.isEmpty()) return;
		--this.size;
	}
	
	public Object peekBack() {
		return this.arr[size-1];
	}
	
	public void remove(Object o) {
		if(!this.contains(o))return;
		int removeLoc = search(o);
		moveLeft(removeLoc);
		--this.size;
	}
	public void removeAll(Object o) {
		if(!this.contains(o))return;
		while(this.contains(o)) {
			int removeLoc = search(o);
			moveLeft(removeLoc);
			--this.size;
		}
	}
	
	public Iterator<T> iterator(){
		return new ListIterator();
	}
	
	@Override
	public String toString() {
		String str="[";
		for(int i=0; i<this.size; ++i) {
			str+=arr[i];
			if(i!=this.size-1) {
				str+=", ";
			}
		}
		return str+"]";
	}
	
	private void increaseCapacity() {
		Object[] arr = new Object[this.capacity*2];
		for(int i=0; i<this.size; ++i) {
			arr[i]=this.arr[i];
		}
		this.arr=arr;
		this.capacity*=2;
	}
	private void moveRight(int index) {
		for(int i=0; i<this.size-index+1; ++i) {
			this.arr[size+1-i]=this.arr[size-i];
		}
	}
	private void moveLeft(int index) {
		for(int i=index; i<this.size-1; ++i) {
			this.arr[i]=this.arr[i+1];
		}
	}
	private int search(Object o) {
		int lo=0;
		int hi=this.size-1;
		while(lo<=hi){
			int mid=lo+(hi-lo)/2;
			if(this.arr[mid]==o) return mid;
			else if(((T)this.arr[mid]).compareTo((T) o)>0){
				if(mid==0)return lo;
				hi=mid-1;
			}
			else lo=mid+1;
		}
		return lo;
	}
}

(search) 

private int search(Object o) {
	int lo=0;
	int hi=this.size-1;
	while(lo<=hi){
		int mid=lo+(hi-lo)/2;
		if(this.arr[mid]==o) return mid;
		else if(((T)this.arr[mid]).compareTo((T) o)>0){
			if(mid==0)return lo;
			hi=mid-1;
		}
		else lo=mid+1;
	}
	return lo;
}

리스트의 특정 요소의 위치를 검색하는 메소드입니다.
정렬 리스트의 경우 요소들이 전부 비교가 가능 해야하기 때문에 T는 비교 가능 즉 comparable이어야 합니다.
각 요소들은 comparable이기 때문에 compareTo를 구현 하고 있어 대소비교가 가능합니다.
그러므로 이제는 선형 검색이 아닌 이진 탐색이 가능해 졌습니다.
이진 탐색이란 아래 그림과 같이 리스트의 처음요소와 끝요소를 각 low와 high로 잡고 가운데 요소 middle과 비교 객체와 비교를 통해 middle이 크면 hight 는 middle-1이 되고 아니면 low가 middle+1이 됩니다.


이 과정을 반복하면 해당값에 가장 가까운 값을 찾을 수 있습니다.
또한 선형 검색과 비교하여 선형 검색의 경우 요소가 n개 이면 n 개를 모두 접근해야 하지만 이진탐색의 경우는 2의 제곱수 만큼 비교 개수가 줄기 때문에 n개가 있다면 log2(n)번이면 모두 접근이 가능합니다.
그렇기 때문에 시간은 O(logn)이 걸리게 됩니다.

(pushFront, pusheBack->add) 

public void add(Object o) {
	if(this.isFull())this.increaseCapacity();
	if(isEmpty() || ((T)this.arr[this.size-1]).compareTo((T)o)<=0)
		this.arr[size]=o;
	else{
		int insertLoc=search(o);
		moveRight(insertLoc);
		arr[insertLoc]=o;
	}
	++size;
}

리스트의 요소를 정렬되게 삽입하는 메소드입니다.
정렬 리스트이기 때문에 pushFront, pusheBack의 메소드가 아닌 add 메소드를 사용해야 합니다.
isFull메소드를 통해 가득차 있으면 increaseCapacity메소드를 실행햐여 용량을 증가 시킵니다.
만약 리스트가 비어 있거나 마즈막 요소보다 크거나 같으면 [size]인덱스에 삽이 후 size를 증가 시킵니다.
위의 경우가 아니면 search메소드를 통해 크기가 같거나 비교객체보다 바로 다음으로 큰 값을 리스트에서 찾아 인덱스를 받아 온후 moveRight메소드를 통해 해당 인덱스 부터 우측으로 1칸씩 이동후 객체를 해당 인덱스에 객체를 삽입하고 size를 증가 시키는 형태입니다.

(생성자) 

public ArrayList(Object... arr){
	this.capacity = arr.length*2;
	this.arr = new Object[this.capacity];
	for(int i=0; i<arr.length; ++i) {
		this.add(arr[i]);
	}
	this.size = arr.length;
}

리스트의 객체를 가변인수로 받는 생성자입니다.
비정렬 리스트와 달리 add메소드를 이용하여 값을 추가하여 생성해 줍니다.

(contains) 

public boolean contains(Object o) {
	int index = search(o);
	if(this.arr[index]==o)return true;
	return false;
}

리스트에 해당 객체가 존재하는지 확인하는 메소드입니다.
search메소드를 이용해 인덱스를 받아와 해당 인덱스와 갑이 같으면 true 없으면 false를 반환하는 형태입니다.

(indexOf) 

public int indexOf(Object o) {
	int index = search(o);
	if(this.arr[index]==o)return index;
	return -1;
}

리스트에 해당 객체가 존재하는 인덱스를 확인하는 메소드입니다.
search메소드를 이용해 인덱스를 받아와 해당 인덱스와 갑이 같으면 해당 인덱스를 없으면 -1을 반환하는 형태입니다.

(remove) 

public void remove(Object o) {
	if(!this.contains(o))return;
	int removeLoc = search(o);
	moveLeft(removeLoc);
	--this.size;
}

리스트의 특정 객체 1개를 삭제하는 메소드입니다.
contains메소드를 이용하여 리스트에 해당 객체가 존재하는지 확인후 없으면 return하여 종료하고 있으면 search메소드를 통해 인덱스를 받아와 해당인덱스 부터 moveLeft메소드를 실행하여 값들을 왼쪽으로 당겨 값을 삭제후 size를 감소하고 종료하는 형태입니다.

(removeAll) 

public void removeAll(Object o) {
	if(!this.contains(o))return;
	while(this.contains(o)) {
		int removeLoc = search(o);
		moveLeft(removeLoc);
		--this.size;
	}
}

리스트의 특정 객체 모두를 삭제하는 메소드입니다.
contains메소드를 이용하여 리스트에 해당 객체가 존재하는지 확인후 없으면 return하여 종료하고 있으면 없을때 까지 반복하여 search메소드를 통해 인덱스를 받아와 해당인덱스 부터 moveLeft메소드를 실행하여 값들을 왼쪽으로 당겨 값을 삭제후 size를 감소시키는 형태입니다.

위의 표와 같이 주요 메소드들의 시간 복잡도를 정리해 보았습니다.
배열기반의 리스트는 확장시 추가 비용이 발생됩니다.
비정렬 기반과 비교하여 보면 contains즉 찾을 때 O(n)->O(logn)으로 바뀌었습니다.
