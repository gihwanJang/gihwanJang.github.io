algorithm Ch.01 : Big-O

<알고리즘>

알리고리즘이란 어떠한 문제를 해결하기 위해 정해진 일련의 절차나 방법입니다.

<알고리즘의 구현>

알고리즘의 구현은 자연어, 의사코드, 순서도, 프로그래밍언어, 인터프리터가 작업하는 제어테이블, 유한상태기계의 상태도 등으로 표현할 수 있지만 해당 post에서는 프로그래밍언어인 java를 통해 서술하도록 하겠습니다.

<알고리즘의 성능 분석>

한 문제를 해결하는 알고리즘은 다양할 수 있습니다.
그러므로 어떤 알고리즘이 이 문제를 해결하는 것에 효과적인지 판단할 수 있어야 합니다.
판단을 하기위해 알고리즘에 대한 성능의 분석은 2가지 측면으로 분석이 가능합니다.
입력시 해당 문제를 해결하는데 걸리는 소요시간을 분석하는 시간 복잡도와 해당 문제를 해결하기 위해 필요한 공간을 분석하는 공간 복잡도가 있습니다.
해당 분석은 (최선), (평균), (최악)의 경우로 분석이 가능하지만 최선의 경우로 분석을 하게 된다면 사용자는 해당 알고리즘을 효율적이라 체감 할 수 없으므로 의미가 없고 평균의 경우 입력값의 확률 알고리즘의 소요시간 등 고려해야할 부분이 너무 많으므로 일반적으로 산술하기 어렵기 때문에 최악의 경우를 보통으로 사용해게 됩니다.
시간 복잡도와 공간 복잡도의 계산은 아래와 같습니다.

  • 시간 복잡도 : 입력에 따라 필요한 기본 연사의 수를 계산합니다.
    기본 연산은 산수, 대입, 비교등이 있으며 각 연산들의 속도는 고려하지 않습니다.
  • 공간 복잡도 : 입력에 대한 필요 공간과 내부적으로 사용하는 메모리 공간을 분석하여 계삽합니다.

<시간 복잡도>

알고리즘의 시간 복잡도를 측정하기 위해 실제 실행 시간을 측정하지 않습니다.
특히, 아래의 이유로 실행 시간을 측정하여 두 알고리즘의 성능을 비교하지 않습니다.

  • 실제 실행 시간을 측정하기 위해서는 구현하여야 한다.
  • 알고리즘의 구현은 개발자에 따라 차이가 있을 수 있고, 실행시간은 언어, 개발환경, 실행환경 등 여러 요인에 의해 차이가 있을 수 있다.
  • 테스트는 다양한 케이스에 대하여 비교하여야 한다.
  • 도출 결과로 부터 서로의 성능 수준을 비교하기 어렵다.

그렇기 때문에 알고리즘은 입력에 대한 기본 연산수를 계산하여 분석합니다.
기본 연산의 성능은 각기 다르지만 같은 문제를 해결하는 알교리즘을 비교하는 것이고, 점근적 분석이기 때문에 결국 비교에 핵심이 되는 연산은 다르지 않기 때문에 문제가 되지 않습니다.

ex) 검색 알고리즘 분석

n개의 정수로 구성된 배열 nums와 특정 정수 v가 주어졌을 때, 해당 배열에 v가 존재하면 true 아니면 false를 반환하는 검색 알고리즘의 시간복잡도를 분석해 보겠습니다. 

	public boolean search(int[] nums, int v) {
		for(int i=0, last=nums.length; i<last; ++i)
			if(nums[i]==v)return true;
		return false;
	}

  • 최선의 경우 : 대입연산 2(i=0,last=nums.length), 비교연산2(i<last, nums[i]==v) -> 4
    (배열의 첫번째 인덱스가 찾는 값인 경우)
  • 최악의 경우 : 대입연산 2(i=0,last=nums.length), 비교연산2n+1, 증감연산 n -> 3n+3
    (배열에 찾는 값이 없는 경우)
  • 평균의 경우 : 배열 내에 있는 값을 검색할 확률, 각위치에 검색 값이 있을 확률 등 여러 수치가 필요하므로 계산하기 어려습니다.

Big-O를 분석하는 방법은 아래와 같습니다.

  • 알고리즘 서술이나 구현 결과를 토대로 최악의 경우 입력 크기 n에 따라 필요한 기본 연산수를 파악하여 T(n)의 다항식을 정의합니다.
  • T(n)의 다항식에서 최고차 항을 제외한 나머지 항은 제고하고, 최고차 항의 계수도 제거합니다.
  • 최종적으로 도출된 다항식이 알고리즘의 Big-O가 됩니다.

위의 검색 알고리즘의 T(n)=3n+3이고 Big-O는 O(n)이 됩니다.

Big-O의 정의는 입력 크기 n에 대해 최악의 경우 알고리즘의 수행 시간을 나타내는 함수 T(n)이 궁극에는 cf(n)보다 항상 증가 속다가 느리면 이 알고리즘의 Big-O는 O(f(n))이라 합니다.
즉 T(n)이 f(n)보다 성능이 같거나 우수하면 O(f(n))이라합니다.
위의 검색 알고리즘의 T(n)=3n+3이고 O(n)입니다.

(빅O, 빅Ω, 빅θ, 리틀o)
보통 알고리즘의 성능 분석 결과를 나타낼 때는 빅O를 가장 많이 사용하지만 빅O 빅θ로 성능을 표현 할 수 있다면 빅θ를 많이 사용합니다.
빅Ω : 입력 크기 n에 대해 최악의 경우 알고리즘의 수행 시간을 나타내는 함수 T(n)이 cf(n)보다 항상 증가 속도가 빠르면 이 알고리즘의 빅Ω는 Ω(f(n))이라 합니다.
빅θ : 특정 알고리즘의 빅O와 빅Ω가 같은경우를 말합니다.
리틀o : 주어진 복잡도보다 무조건 빠른 것을 말합니다.

빅O를 분석 할 때 입력 크기에 대한 함수를 정의해야 합니다.
이때 입력 크기는 n개의 데이터를 처리하면 입력 크기는 n이라고 표현합니다.
즉 선형검색의 경우 검색하는 배열의 크기가 입력의 크기가 됩니다.
하지만 정확한 입력 크기는 입력을 표현하기 위해 필요한 문자의 수(비트)입니다.

ex) 소수 판별

양의 정수 x가 소수면 true 아니면 false를 출력합니다. 

	public boolean search(int x) {
		for(int i=2; i<Math.sqrt(x); ++i)
			if(x%i==0)return false;
		return true;
	}

이 알고리즘은 최악의 경우 √ x -1만큼 나머지 연산과 비교 연산을 수행하여야 합니다.
즉 2(√ x -1)비용이 소요되지만 이 알고리즘의 빅O는 O(n1/2)가 아닙니다.
하나의 정수를 표현하기 위한 비트수는 log2x 가 필요합니다.
즉 x=2n이므로 식에 x대신 대입해주면 빅O는 O(2n)이됩니다.
이처럼 입력의 크기를 정확하게 분석하지 못하고 주어진 정수의 값에 따라 비용이 달라지는 경우 위와 같이 분석해야 합니다.

위에서는 하느의 변수만을 사용하였습니다.
하지만 알고리즘을 분석하다 보면 여러 개의 변수를 이용하여 빅O를 표현해야하는 경우도 있습니다.
가장 대표적으로 그래프 알고리즘을 예로 들 수 있습니다.
그래프의 경우 노드 수와 간선의 수를 이용하여 성능을 표현하기에 2개의 변수로 표현하는 경우가 많습니다.
하지만 다중 변수로 표현되면 실제 각 변수들의 관계와 입력을 표현하는 방법에 따라 성능 수준이 결정 되기 때문에 빅O만을 보고 해당 알고리즘의 성능 수준을 단정 지을 수 없습니다.
다만 알고리즘은 같은 문제를 해결하는 것을 비교하는게 목적이므로 다중 변수를 이용한 시간복잡도들이 주어 졌을때 어느 것이 더 우수한지는 쉽게 판단할 수 있습니다.
또한 각 변수가 알고리즘에 어떻게 영향을 주는지 알 수 있기 때문에 관계성을 이용하여 단일 변수로 사용하는 것 보다 다중 변수 빅O를 그대로 사용하는 것이 더 좋습니다.

<공간 복잡도>

공간 복잡도는 주어진 입력의 크기와 알고리즘 내부적으로 사용되는 메모리 공간에 의해 결정 됩니다.
하지만 보통 같은 문제를 비교하기에 입력크기보다는 알고리즘 내부적으로 사용하는 메모리 공간을 더 중요시합니다.
재귀 호출을 이용하는 알고리즘은 재귀 호출의 깊이만큼 함수 스택 공간을 사용합니다.
그러므로 (재귀 호출의 최대 깊이 x 함수 스택 공간의 크기)를 공간 복잡도에 포함해야 합니다.
공간복잡도의 분석은 알고리즘의 수행시 사용되는 전체 공간이 아닌 가장 많은 공간을 사용했을 때의 공간크기를 고려해 공간 복잡도를 분석합니다.
