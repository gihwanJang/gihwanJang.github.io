6. 비정렬 단일 연결 리스트(UnsortedSingleLinkedList)

지금까지는 배열을 이용한 리스트를 post해왔습니다.
이번 post부터는 배열리스트가 아닌 연결리스트에 대해 post해 보려고 합니다.
연결리스트는 배열리스트 처럼 데이터를 저장하는 일반 자료형를 사용하지 않고 데이터를 담고있는 Node를 내부 클래스로 별도로 만들어 사용합니다.
해당 Node의 구성에 따라 연결 리스트의 종류가 나누어 집니다.
Node가 데이터인 item과 다음 노드 next만을 가지고 있으면 다음 노드 하나만으로 연결 되기 때문에 단일 연결구조로 분류 되고, Node가 item, next 이전 노드 prev를 가지고 있으면 다음과 이전 두개로 연결 되기 때문에 이중 연결 리스트로 분류 됩니다.
이외에도 연결리스트는 Node의 연결에 따라 다른 분류로 분류되지만 단일과 이중 연결만을 다루도록 하겠습니다.
배열기반의 리스트가 있음에도 연결 리스트를 사용하는 이유는 연결 리스트가 가지는 특정 장점이 있기 때문입니다.
<연결 리스트의 특징> 1. 용량의 제약이 없습니다.
-> 배열기반 리스트의 경우에는 처음 지정한 용량을 초과하게 되면 2배의 용량이 되는 배열을 새로 만들어 해당 내용들을 옮겨 새로 만들어 주어야 합니다.
반면 연결리스트의 경우에는 정해진 용량이 없고 노드를 추가 삭제하여 사용하는 것이기 때문에 공간 부족이나 낭비가 발생하지 않습니다.
그렇게 때문에 리스트를 사용 할때 이 부분을 고려하여 만약 자신이 사용할 데이터가 용량의 범위가 크게 유동적이거나 미리 알 수 없다면 연결 리스트를 데이터가 정적이고 용량이 딱 정해져 있다면 배열리스트를 사용하는 것이 좋습니다.
2. 데이터의 삽입과 삭제가 쉽습니다. ->배열기반 리스트의 경우 인덱스가 정해져 있고 해당 인덱스를 이용하여 접근해 데이터를 가져오기 때문에 데이터 삽입삭제시 해당 인덱스가 비어있지 않도록 매꾸어주어야 하는데 해당 과정에서 비용이 듭니다.
반면 연결리스트의 경우 next혹은 prev즉 연결만 바꾸어주면 되기에 삽입과 삭제가 쉽습니다. 3. 순차접근을 해야합니다. ->배열기반 리스트의 경우 인덱스가 있어 임의 접근이 가능합니다.
반면 연결리스트의 경우 인덱스가 아닌 연결을 따라 접근하는 순차접근만이 가능합니다. 4. 운영체제의 캐시 지역성을 활용 할 수 없습니다. ->배열기반 리스트의 경우 연속된 공간을 받아 오기 때문에 cache locality을 활용하여 성능이 좋습니다.
반면 연결리스트의 경우 연속된 공간이 아니기 때문에 cache locality을 활용 할 수 없습니다.
그러므로 성능면에서는 배열기반 리스트에 비해 좋지 않습니다.

(코드) 

import java.util.Iterator;
import java.util.Objects;

class SingleUnsortedLinkedList<E> implements Iterable<E>, Cloneable{
	private static class Node{
		private Object item;
		private Node next=null;
		private Node(Object value) {
			this.item=value;
		}
		public Node clone() throws CloneNotSupportedException{
			Node cloned = (Node)super.clone();
			return cloned;
		}
	}
	private Node head = null;
	private int size = 0;
	
	private class LinkedListIterator implements Iterator<E>{
		private  int cursor =0;
		private Node curr=head;
		@Override
		public boolean hasNext() {
			return cursor<size;
		}
		@Override
		public E next() {
			Object val=curr.item;
			++cursor;
			curr=curr.next;
			return (E)val;
		}
	}
	
	@Override
	public String toString() {
		String str="[";
		Node curr=head;
		for(int i=0;i<size;++i) {
			str+=curr.item;
			curr=curr.next;
			if(i!=this.size-1) {
				str+=", ";
			}
		}
		return str+"]";
	}
	
	public SingleUnsortedLinkedList(){
		this.head = null;
		this.size = 0;
	}
	
	public SingleUnsortedLinkedList(Object... arr){
		for(int i=arr.length-1;i>=0;--i) {
			this.pushFront(arr[i]);
		}
	}
	
	public boolean isEmpty() {
		if(this.size==0)return true;
		return false;
	}
	
	public int size() {
		return this.size;
	}
	
	public void clear() {
		this.size=0;
		this.head=null;
	}
	
	public int indexOf(Object o) {
		if(this.isEmpty())return -1;
		Node curr=head;
		for(int i=0;i<size;++i) {
			if(curr.item==o)return i;
			curr=curr.next;
		}
		return -1;
	}
	
	public boolean contains(Object o) {
		if(this.isEmpty())return false;
		return this.indexOf(o)!=-1?true:false;
	}
	
	public Object get(int index) {
		Objects.checkIndex(index, this.size-1);
		Node curr=this.head;
		for(int i=0;i<index;++i) curr=curr.next;
		return curr.item;
	}
	
	public void pushFront(Object val) {
		Node newNode = new Node(val);
		newNode.next=this.head;
		this.head=newNode;
		++this.size;
	}
	
	public void popFront() {
		if(this.isEmpty())return;
		this.head=head.next;
		--this.size;
	}

	public Object peekFront() {
		return head.item;
	}
	
	public void pushBack(Object val) {
		Node newNode = new Node(val);
		Node curr=this.head;
		for(int i=1; i<size; ++i) curr=curr.next;
		curr.next=newNode;
		++this.size;
	}
	
	public void popBack() {
		if(this.isEmpty())return;
		Node curr=this.head;
		for(int i=2; i<size; ++i) curr=curr.next;
		curr.next=null;
		--this.size;
	}
	
	public Object peekBack() {
		Node curr=head;
		for(int i=1; i<size; ++i) curr=curr.next;
		return curr.item;
	}
	
	public void remove(Object val) {
		if(this.indexOf(val)==-1)return;
		Node curr=this.head;
		Node prev=new Node(null);
		prev.next=this.head;
		while(curr.item!=val) {
			curr=curr.next;
			prev=prev.next;
		}
		if(curr==this.head)this.head=this.head.next;
		else prev.next=curr.next;
		--this.size;
	}
	
	public void removeAll(Object val) {
		if(this.indexOf(val)==-1)return;
		Node curr=this.head;
		Node prev=new Node(null);
		prev.next=this.head;
		while(curr!=null) {
			if(curr.item!=val) {
				curr=curr.next;
				prev=prev.next;
			}
			else {
				if(curr==this.head)this.head=this.head.next;
				else prev.next=curr.next;
				curr=curr.next;
				--this.size;
			}
		}
	}
	
	public Iterator<E> iterator(){
		return new LinkedListIterator();
	}
	
	public SingleUnsortedLinkedList<E> clone() throws CloneNotSupportedException{
		SingleUnsortedLinkedList<E> cloned = (SingleUnsortedLinkedList<E>)super.clone();
		Node curr=cloned.head;
		for(E i:this) {
			Node newNode = new Node(i);
			curr=newNode;
			curr=curr.next;
		}
		return cloned;
	}
}

		private Node next=null;
		private Node(Object value) {
			this.item=value;
		}
		public Node clone() throws CloneNotSupportedException{
			Node cloned = (Node)super.clone();
			return cloned;
		}
	}

데이터와 연결정보를 저장할 노드 클래스입니다.
item은 실질적인 데이터를 저장합니다.
next는 다음 노드의 정보를 저장합니다.
해당 노드의 생성자는 값을 받아 노드를 생성하게 됩니다.
clone 메소드는 해당 연결리스트의 clone을 만들때 해당 메소드를 사용하여야하기 때문에 구현하였습니다.
