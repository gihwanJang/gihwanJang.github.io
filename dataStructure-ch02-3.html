자료구조 Ch.02 : 리스트

5. 순환 배열기반 비정렬 리스트(SortedArrayList)

앞에서 봤던 배열기반 비정렬 리스트의 경우 pushFront, popFront의 경우 O(n)의 시간복잡도를 가졌지만 순환 배열을 이용할 경우 이것을 O(n)으로 개선 할 수 있습니다.
기존 배열의 경우 앞의 index는 0으로 고정이 되어 있지만 순환배열의 경우 앞의 index가 0으로 고정되어 있지 않고 조작에 따라 해당 index는 유동적으로 변하게 됩니다.
쉽게 배열의 끝과 처음을 붙였다고 생각하면 좋을 것 같습니다.


해당 모습은 위 그림과 같고 이때 처음을 head라고 할때 이 head가 B를 가르키게 되면 처음은 B가 되게 됩니다.
즉 리스트는 B,C,D,F를 저장하고 있게 됩니다.

(코드) 

import java.util.Iterator;

class ArrayList<T> implements Iterable<T>, Cloneable{
	private Object[] arr = null;
	private int capacity;
	private int size;
	private int head = 0;
	private int tail = 0;
	
	private class ListIterator implements Iterator<T>{
		private int cursor =head;
		private int count =0;
		@Override
		public boolean hasNext() {
			return count<size;
		}
		@Override
		public T next() {
			++count;
			int item=cursor;
			cursor=(cursor+1)%capacity;
			return (T)arr[item];
		}
	}
	
	public ArrayList(){
		this.capacity = 10;
		this.arr = new Object[this.capacity];
		this.size = 0;
	}
	public ArrayList(Object... arr){
		this.capacity = arr.length*2;
		this.arr = new Object[this.capacity];
		for(int i=0; i<arr.length; ++i) {
			this.arr[i]=arr[i];
		}
		this.size = arr.length;
		this.tail = (size-1) % capacity;
	}
	
	public boolean isEmpty() {
		if(this.size==0)return true;
		return false;
	}
	
	public boolean isFull() {
		if(this.size==this.capacity)return true;
		return false;
	}
	
	public int size() {
		return this.size;
	}
	
	public void clear() {
		this.size=0;
		this.head=0;
		this.tail=0;
	}
	
	public boolean contains(Object o) {
		if(this.search(o,this.head)!=-1)
			return true;
		return false;
	}
	
	public int indexOf(Object o) {
		return this.search(o,this.head);
	}
	
	public void pushFront(Object o) {
		if(this.isFull())this.increaseCapacity();
		this.arr[(this.head+this.capacity-1)%this.capacity]=o;
		this.head=(this.head+this.capacity-1)%this.capacity;
		++this.size;
	}
	
	public void popFront() {
		if(this.isEmpty()) return;
		this.head=(this.head+1)%this.capacity;
		--this.size;
	}
	
	public Object peekFront() {
		return this.arr[this.head];
	}
	
	public void pushBack(Object o) {
		if(this.isFull())this.increaseCapacity();
		this.arr[(this.tail+1)%this.capacity]=o;
		this.tail=(this.tail+1)%this.capacity;
		++this.size;
	}
	
	public void popBack() {
		if(this.isEmpty()) return;
		this.tail=(this.tail+this.capacity-1)%this.capacity;
		--this.size;
	}
	
	public Object peekBack() {
		return this.arr[tail];
	}
	
	public void remove(Object o) {
		if(this.search(o,head)==-1)return;
		int idx=head;
		while(true){
			if(this.arr[idx]==o) {
				this.moveLeft(idx);
				--this.size;
				break;
			}
			idx=(idx+1)%this.capacity;
		}
	}
	public void removeAll(Object o) {
		if(this.search(o,head)==-1)return;
		int idx=head;
		while(true){
			if(this.arr[idx]==o) {
				this.moveLeft(idx);
				--this.size;
				if(this.search(o,idx)==-1)break;
				else continue;
			}
			idx=(idx+1)%this.capacity;
		}
	}
	
	public Iterator<T> iterator(){
		return new ListIterator();
	}
	
	@Override
	public String toString() {
		String str="[";
		for(int i=this.head; i!=this.tail; i=(i+1)%capacity) {
			str+=arr[i];
			str+=", ";
		}
		str+=arr[this.tail];
		return str+"]";
	}
	
	public ArrayList<T> clone() throws CloneNotSupportedException{
		ArrayList<T> cloned = (ArrayList<T>)super.clone();
		cloned.arr=(Object[])arr.clone();
		return cloned;
	}
	
	private void increaseCapacity() {
		Object[] arr = new Object[this.capacity*2];
		int idx=this.head;
		int newhead=0;
		while(true){
			arr[newhead++]=this.arr[idx];
			if(idx==tail)break;
			idx=(idx+1)%this.capacity;
		}
		this.arr=arr;
		this.head=0;
		this.tail=this.size-1;
		this.capacity*=2;
	}
	private void moveLeft(int index) {
		while(true){
			this.arr[index]=this.arr[(index+1)%capacity];
			if(index==this.tail)break;
			index=(index+1)%this.capacity;
		}
		this.tail=(this.tail+this.capacity-1)%this.capacity;
	}
	private int search(Object o, int idx) {
		if(isEmpty()) return -1;
		while(true){
			if(this.arr[idx]==o) return idx;
			if(idx==this.tail)break;
			idx=(idx+1)%this.capacity;
		}
		return -1;
	}
}

(멤버 변수) 

private Object[] arr = null;
private int capacity;
private int size;
private int head = 0;
private int tail = 0;

맴버 변수는 3가지 arr, capacity, size에서 head와 tail을 추가로 설정을 하였습니다.
head : 리스트에서 시작 index를 저정하는 변수입니다. tail : 리스트에서 끝 index를 저장하는 변수입니다.

(ListIterator) 

	private class ListIterator implements Iterator<T>{
		private int cursor =head;
		private int count =0;
		@Override
		public boolean hasNext() {
			return count<size;
		}
		@Override
		public T next() {
			++count;
			int item=cursor;
			cursor=(cursor+1)%capacity;
			return (T)arr[item];
		}
	}

리스트의 Iterator로서 순차접근을 하게 해주는 내부 클래스입니다.
앞의 배열을 이용한 리스트의 경우 기존 방식대로 순차접근을 통해 접근을 하면 되지만 해당 순환배열의 경우 처음과 끝이 정해져 있지 않기 때문에 head와 tail을 이용하여 Iterator을 재정의해 주었습니다.

(생성자) 

public ArrayList(Object... arr){
	this.capacity = arr.length*2;
	this.arr = new Object[this.capacity];

}

앞의 생성자와 거의 동일하지만 마지막에 tail의 값을 지정해 주는 것의 차이를 볼 수 있습니다.

(claer) 

public void clear() {
	this.size=0;
	this.head=0;
	this.tail=0;
}

리스트를 초기화 해주는 메소드입니다.
앞의 clear의 경우 size만을 0으로 초기화 시켜 줬지만 순환배열을 이용할 경우 head와 tail모두 0으로 초기화 주어야 합니다.

(search) 

	private int search(Object o, int idx) {
		if(isEmpty()) return -1;
		while(true){
			if(this.arr[idx]==o) return idx;
			if(idx==this.tail)break;
			idx=(idx+1)%this.capacity;
		}
		return -1;
	}

가장 아래에 위치해 있는 메소드로 리스트에서 매개변수로 받은 인덱스부터 받은 객체와 동일한 것이 있는지 찾는 메소드입니다.
많은 메소드가 해당 메소드를 이용하기 때문에 먼저 소개해 드리겠습니다.
해당 메소드는 while문을 통해 반복하도록 하였고 매개변수로 받은 인덱스부터 받은 객체와 동일 한지 확인하며 해당인덱스는 +1을 하며 증가하게 됩니다.
이때 배열의 인덱스가 capacity를 넘어 갈 수 있기 때문에 capacity로 나머지 연산을 하여 처음으로 돌아가게 하여 tail까지 선형검색을 하고 있으면 해당 인덱스를 없으면 -1을 받환 하게 합니다.

(contains) 

public boolean contains(Object o) {
	if(this.search(o,this.head)!=-1)
		return true;
	return false;
}

리스트의 특정 객체가 있는지 확인하는 메소드입니다.
search 메소드를 이용하여 배열의 처음 즉 head부터 tail 까지 해당 객체가 있는지 검색후 있으면 true 없으면 false를 반환합니다.

(pushFront) 

public void pushFront(Object o) {
	if(this.isFull())this.increaseCapacity();
	this.arr[(this.head+this.capacity-1)%this.capacity]=o;
	this.head=(this.head+this.capacity-1)%this.capacity;
	++this.size;
}

리스트 맨 앞에 객체를 넣는 메소드입니다.
앞에서본 pushFront의 경우 빈 공간이 있으면 moveRight라는 메소드를 통해 맨 앞에서 부터 오른쪽으로 한칸씩 옮긴 뒤 맨앞에 객체를 넣고 size를 증가시키는 형태였지만 순환 배열을 이용한 리스트에서는 head의 인덱스를 감소시켜 삽입하기 때문에 시간복잡도 O(1)에 할 수 있습니다.
이때 리스트의 인덱스를 감소 시킬때 해당 인덱스가 음수가 되는 것을 방지하기 위하여 현재 head 값에 capacity를 더한후 -1연산후 capacity로 나머지 연산을 합니다.
이렇게 하면 예를 들어 인덱스가 0일때 (0+capacity-1)%capacity=capacity-1로 배열의 가장 끝이 리스트의 처음이 됩니다.

(popFront) 

public void popFront() {
	if(this.isEmpty()) return;
	this.head=(this.head+1)%this.capacity;
	--this.size;
}

배열 맨 앞에 객체를 빼는 메소드입니다.
앞의 리스트에서는 맨앞에 객체가 있으면 moveLeft라는 메소드를 통해 앞의 요소를 뒤에 있는 요소로 덮어 쓴후 size를 감소시키는 형태였지만 순환 배열에서는 head의 위치만 +1연산을 하면 되기 때문에 시간복잡도 O(1)에 가능합니다.
이때 head에 +1연산에는 음수가 될 가능성이 없기 때문에 그냥 +1연산후 capacity로 나머지연산을 하여 head의 위치를 정해줍니다.

(pushBack) 

public void pushBack(Object o) {
	if(this.isFull())this.increaseCapacity();
	this.arr[(this.tail+1)%this.capacity]=o;
	this.tail=(this.tail+1)%this.capacity;
	++this.size;
}

배열 맨 뒤에 객체를 넣는 메소드입니다.
앞의 리스트에서는 빈 공간이 있으면 맨뒤 다음 인덱스(index : size)에 객체를 넣고 size를 증가시키는 형태였지만 순환배열을 이용했을 경우 tail의 맴버변수가 추가 되기 때문에 tail을 증가 시켜준후 해당 tail에 객체를 삽입합니다.
pushBack의 맨뒤의 다음 인덱스에 접근하여 값을 삽입하기 때문에 O(1)의 실행 시간이 걸립니다.

(popBack) 

public void popBack() {
	if(this.isEmpty()) return;
	this.tail=(this.tail+this.capacity-1)%this.capacity;
	--this.size;
}

배열 맨 뒤의 객체를 빼는 메소드입니다.
앞의 리스트의 메소드에서 tail을 감소시키는 코드만 추가됩니다.
이때 tail의 감소시 capacity를 더해주는 것에 유의해야합니다.
popBack의 경우 size,tail 값만 감소 시키기 때문에 O(1)의 실행 시간이 걸립니다.

(remove) 

public void remove(Object o) {
	if(this.search(o,head)==-1)return;
	int idx=head;
	while(true){
		if(this.arr[idx]==o) {
			this.moveLeft(idx);
			--this.size;
			break;

	}

리스트의 특정 객체 1개를 삭제하는 메소드입니다.
search메소드를 이용하여 리스트에 해당 객체가 존재하는지 확인후 없으면 return하여 종료하고 있으면 선형검색을 통해 해당 인덱스 부터 moveLeft메소드를 실행하여 값들을 왼쪽으로 당겨 값을 삭제후 size를 감소하고 종료하는 형태입니다.
해당 객체가 있는 곳까지 선형으로 검색하기 때문에 O(n)의 실행 시간이 걸립니다.
